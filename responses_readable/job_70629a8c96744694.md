Short answer: not in a single commit operation, but you can achieve the effect of triggering multiple GitHub Actions pipelines at once with a few practical approaches.

Why you can’t “commit to multiple branches” directly

In Git, a commit is tied to a specific branch (technically a ref). You can apply the same changes to multiple branches, but each branch still gets its own commit.

Practical ways to trigger pipelines on multiple branches
1. Push the same commit to multiple branches

If the branches can point to the same commit:

Bash
git commit -m "Your change"
git push origin HEAD:branch1 HEAD:branch2 HEAD:branch3

This pushes the same commit to multiple branches in one command.
✅ All associated GitHub Actions workflows for those branches will trigger simultaneously.

2. Cherry-pick or merge into multiple branches

If branches differ:

Bash
git checkout branch1
git commit -m "change"
git push

git checkout branch2
git cherry-pick branch1
git push

You can script this to make it fast. Each push triggers its own workflow.

3. Use a script to automate multi-branch updates

For example:

Bash
for branch in branch1 branch2 branch3; do
  git checkout $branch
  git merge main
  git push origin $branch &
done
wait

The & runs pushes in parallel → pipelines start nearly simultaneously.

4. Trigger workflows without pushing commits (recommended for CI control)

If your goal is just to run pipelines, not change code:

a. workflow_dispatch

Trigger workflows manually or via API for multiple branches.

b. GitHub CLI
Bash
gh workflow run your-workflow.yml --ref branch1 &
gh workflow run your-workflow.yml --ref branch2 &
wait
c. Repository dispatch (API)

You can fire multiple events programmatically to different branches.

5. Use a matrix strategy instead (often better)

Instead of multiple branches, design one workflow:

YAML
strategy:
  matrix:
    branch: [branch1, branch2, branch3]

This runs jobs in parallel inside a single workflow run.

Key takeaway

❌ Git doesn’t support “one commit → multiple branches” as a single atomic action

✅ You can push the same commit to multiple branches at once

✅ Or trigger workflows directly without commits (often cleaner)
