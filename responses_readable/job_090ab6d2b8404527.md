Got it! You want a script that can commit changes across multiple branches in a Git repository so that each branch triggers its own GitHub Actions pipeline simultaneously. I’ll assume the repo is local and you want a clean, repeatable process. Here's a robust Bash script for this purpose:

Bash
#!/bin/bash

# ===============================================
# Script: multi-branch-commit.sh
# Purpose: Commit and push changes to multiple branches
#          so GitHub Actions pipelines run concurrently
# Usage: ./multi-branch-commit.sh "Commit message"
# ===============================================

# Check if commit message is provided
if [ -z "$1" ]; then
  echo "Usage: $0 \"Commit message\""
  exit 1
fi

COMMIT_MSG="$1"

# List of branches you want to commit to
BRANCHES=("main" "develop" "feature-1" "feature-2")  # modify as needed

# Function to commit and push to a branch
commit_branch() {
  local branch=$1
  echo "Processing branch: $branch"
  
  # Checkout the branch
  git checkout "$branch" || { echo "Failed to checkout $branch"; return 1; }
  
  # Add all changes
  git add . || { echo "Failed to add changes"; return 1; }
  
  # Commit changes
  git commit -m "$COMMIT_MSG" || echo "No changes to commit on $branch"
  
  # Push changes
  git push origin "$branch" || { echo "Failed to push $branch"; return 1; }
  
  echo "✅ Changes pushed to $branch"
}

# Loop over branches in background to run pipelines simultaneously
for branch in "${BRANCHES[@]}"; do
  commit_branch "$branch" &
done

# Wait for all background jobs to finish
wait

echo "All branches processed."
How it works:

You provide a commit message when running the script.

It loops through each branch in the BRANCHES array.

For each branch:

Checks it out.

Stages all changes.

Commits if there are any changes.

Pushes to the remote repo.

Each branch is processed in the background (&) so the pushes happen concurrently, triggering GitHub Actions pipelines simultaneously.

wait ensures the script waits until all background jobs finish before exiting.

⚡ Tips:

Make sure your local branches are up-to-date before running this to avoid conflicts.

If a branch has no changes, it will skip the commit but still continue processing others.

You can add more branches to the BRANCHES array easily.
