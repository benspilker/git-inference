The ARI script runs in Windmill, which enforces multiple types of timeouts:

Per-step timeout: Approximately 4 minutes (~240 seconds). Logs may indicate this via messages like [windmill] Previous logs have been saved.

Total script timeout: Approximately 14 minutes (~840 seconds). This may differ across workers depending on configuration.

Variable or inconsistent timeouts: Elapsed times not exactly matching the above may indicate different worker configurations, memory pressure, or resource limits.

Common indications of timeout:

Exit code -9 (SIGKILL) is a strong indicator that the process was forcefully killed.

Output may be truncated or abruptly end without "Processing Phase Finished".

Log messages like Creating Job: indicate processing started but may not have completed.

Recommendations:

Break scripts into multiple Windmill steps or use a Flow.

Check and potentially increase per-step and total script timeouts in Windmill settings.

Ensure workers have sufficient memory and CPU allocation.

ARI has two main phases:

Extraction Phase – Collects Azure resource data from APIs.

Completion is confirmed by a line like "Extraction Phase Finished: HH:MM:SS:MS".

Processing Phase – Generates the Excel report.

Completion is confirmed by the line "Processing Phase Finished".

Observations:

Extraction may succeed but processing may fail if timeouts or resource limits are exceeded.

Excel file generation happens only during the processing phase, so failures here result in no output.
