Overview

This Python codebase is designed to generate a full Azure Resource Inventory – Special Edition (ARI-SE) v2.0 HTML report. It ingests structured data (via a context object) and produces a rich, interactive HTML dashboard showing Azure health, cost, and improvement opportunities. The report includes KPIs, advisor findings, risk analyses, red-flagged items, cost savings, and projected health improvements.

The framework is modular, separating report data modeling, section HTML generation, and template rendering, with a CLI for easy execution.

Key Components
1. HTML Section Builders

Each _build_*_section_html function converts a slice of report_data into a fully formatted HTML section:

_build_redflags_section_html → Highlights resources with critical or high-severity issues, collapsible for detail.

_build_cost_section_html → Shows monthly/annual savings opportunities, reservation recommendations, and cost notes.

_build_findings_section_html → Categorizes advisor findings and highlights unique high-impact issues.

_build_changes_section_html → Tracks changes since last review; defaults to a “baseline established” message if no historical data.

_build_improvements_section_html → Displays projected health score improvements and prioritization of actions.

_build_score_method_section_html → Explains health score calculation across Security, Reliability, Governance, Cost, and Operations.

Other sections include subscriptions, governance, risks, actions, hygiene, and expiring credentials.

Example Usage

Generate a report from JSON model:

Bash
python generate_ari_se_v2.py --model-json model_data.json --output reports/ari_report.html

Emit a starter model template for editing:

Bash
python generate_ari_se_v2.py --emit-model-template

Render using a custom template:

Bash
python generate_ari_se_v2.py --template templates/custom_template.html --output reports/custom_report.html
